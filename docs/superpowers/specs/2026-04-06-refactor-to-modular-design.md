# Refactor to Modular Source with Vite Build

**Date:** April 6, 2026  
**Project:** GitHub Copilot Quota Planner  
**Scope:** Split single `index.html` (1361 lines) into modular source files, add Vite build with inline plugin, and establish testable unit boundaries.

## 1. Background

The current app is a single 1361-line `index.html` file containing all HTML, CSS, and JavaScript logic. While this works for zero-dependency distribution (open directly in browser), the monolithic structure:
- Makes file navigation difficult for future enhancements
- Provides no natural test boundaries
- Lacks a development workflow (no HMR, no dev server)

**Goal:** Maintain the "single file, no dependencies" distribution guarantee while enabling modular development, automated testing, and a better dev experience.

## 2. Approach: Vite + vite-plugin-singlefile

Build with **Vite** (standard, lightweight bundler) and **vite-plugin-singlefile** (inlines CSS + JS into the output `index.html`). Output remains:
- вњ… Single `index.html` file
- вњ… Non-minified for full transparency
- вњ… No external dependencies (only build-time devDependencies)
- вњ… No framework overhead

**Why not alternatives:**
- Nuxt/Vue: adds framework bytecode to final output, violates transparency goal
- Custom Node script: reinvents Vite, loses HMR, less maintainable
- No build: stays monolithic, loses dev experience

## 4. Module Responsibilities

### `auth.js` вЂ” GitHub Token & OAuth Management
**Purpose:** Handle GitHub authentication, token storage, user profile fetching.

**Public exports:**
- `getToken()` в†’ token string or null
- `getUser()` в†’ `{ login, name, avatar_url }` or null
- `ghHeaders()` в†’ object with Authorization and Accept headers
- `renderAuthCard()` в†’ render "Connect" or "Signed in as..." UI
- `openAuthModal()` в†’ show token input modal
- `closeAuthModal()` в†’ hide modal
- `signOut()` в†’ clear localStorage and UI

**State:** localStorage keys `gh_token`, `gh_user`

**API calls:** 
- `GET /user` (for token validation)
- `GET /copilot_internal/user` (for quota data, in `main.js`)

**Tests:** Token format validation, localStorage read/write, API header construction, fallback behavior on 404 from copilot_internal.

---

### `calendar.js` вЂ” Day-Off Selection & Weekend Exclusion
**Purpose:** Manage calendar UI for marking custom day-offs and toggling weekend exclusion.

**Public exports:**
- `openCalendar()` в†’ show calendar popup
- `closeCalendar()` в†’ hide and sync dayoffs to UI
- `renderCalendar()` в†’ draw month grid with navigation
- `calToggleDay(iso)` в†’ toggle ISO date string in/out of dayoffs Set
- `clearCustomDayoffs()` в†’ reset all custom marks and re-render
- `countEffectiveDayoffs()` в†’ count dayoffs that don't overlap with excluded weekends
- `getExcludeWeekends()` в†’ read checkbox state

**State:** 
- `calCustomDayoffs` (Set of ISO date strings like "2026-04-06")
- `calViewYear`, `calViewMonth` (for popup navigation, 0-indexed month)

**Tests:** Weekend counting, toggle mechanics, dayoff counting (no double-count with weekends), calendar grid correctness (today highlighted, weekends styled).

---

### `budgetCalculator.js` вЂ” Core Quota Math (Pure, Testable)
**Purpose:** Calculate all budget metrics. **No DOM dependenciesвЂ”pure functions only.**

**Public exports:**
- `calculateBudget(usage%, currentDay, totalDaysInMonth, excludeWeekends, customDayoffSet)` в†’ returns object:
  ```js
  {
    remaining%,
    remaining requests,
    daysLeft (calendar),
    workingDaysLeft,
    workingDaysElapsed,
    idealDaily%,
    idealDaily requests,
    burnRate%,
    burnRate requests,
    projected% (end of month),
    projected requests,
    perfectTarget% (uniform pace),
    perfectTarget requests,
    vsTarget% (current vs. target),
    vsTarget requests,
    paceRatio (burnRate / idealDaily),
    statusColor,
    paceText
  }
  ```

**Tests:**
- Edge cases: day 1, day 15, day 31 of month
- Extreme usage: 0%, 50%, 100%
- Weekends on/off
- Custom dayoffs (various counts)
- Zero working days remaining (should not divide by zero)
- Perfect pace calculation (uniform burn across working days)

---

### `uiHelpers.js` вЂ” DOM Manipulation & Formatters
**Purpose:** Update DOM state, sync inputs/sliders, format numbers, bind common events.

**Public exports:**
- `syncUsage(val)` в†’ update slider + input + display from slider change
- `syncUsageFromInput(val)` в†’ update slider + input + display from number input
- `updateStatus()` в†’ recalculate via `budgetCalculator`, update all stat displays
- `updateRequestsToday(idealDay)` в†’ populate "Requests You Can Make Today" grid
- `renderAllMonths()` в†’ populate model budget reference table
- `stepNum(id, delta, min, max)` в†’ increment/decrement spinner inputs
- `escHtml(str)` в†’ escape `&<>"` for safe innerHTML injection
- Formatters: `fmt1(n)`, `fmt2(n)`, `fmtInt(n)`, `barColor(pct)`

**Tests:** Formatter correctness, sync behavior (verify input/slider/display stay in sync), HTML escaping, DOM element updates.

---

### `main.js` вЂ” Entry Point & Orchestration
**Purpose:** Initialize app on load, wire up event listeners, fetch quota on launch.

**Responsibilities:**
1. Check localStorage for saved token
2. Call `auth.renderAuthCard()` (draw initial auth UI)
3. If token exists, call `fetchRealUsage()` (silent API fetch)
4. Set initial input values (usage 50%, day = today, month length = current)
5. Bind all event listeners (sliders, buttons, calendar opens, etc.)
6. Call `updateStatus()` to render initial budget cards

**No exportsвЂ”orchestration only.**

**Global state (read/modified by other modules):**
- `quotaEntitlement`, `quotaRemaining` (set by `fetchRealUsage`, read by budget calc)
- `calCustomDayoffs` (Set, shared with calendar.js and budgetCalculator.js)
- `calViewYear`, `calViewMonth` (owned by calendar.js)

## 5. Build & Development Workflow

### Scripts (package.json)
```json
{
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview",
    "test": "vitest",
    "test:ui": "vitest --ui"
  }
}
```

### Development
- `npm run dev` в†’ Vite dev server on `http://localhost:5173` with HMR
- Edit files in `src/`, save, see changes instantly in browser
- Full source maps for debugging

### Production Build
- `npm run build` в†’ Vite + vite-plugin-singlefile
- Output: `dist/index.html` (single file, CSS + JS inlined)
- Minification: **disabled** (via `build.minify: false` in vite.config.js) for transparency
- Result: fully transparent, readable vanilla JS (no uglification)

### Testing
- `npm run test` в†’ Vitest in watch mode
- `npm run test:ui` в†’ Vitest dashboard (optional, for visual feedback)
- All unit tests pass before merge

## 6. Testing Strategy

### Test Organization
- Tests live in `src/tests/` alongside source modules
- Each domain gets its own test file

### `budgetCalculator.test.js`
**Tests:** Pure math functionsвЂ”no DOM, deterministic inputs/outputs.
- Scenarios: day 1/15/31, usage 0/50/100%, weekends on/off, 0-3 custom dayoffs
- Edge cases: no days remaining, zero working days, perfect pace accuracy
- Expected: all tests pass in <100ms total

### `auth.test.js`
**Tests:** Token validation, localStorage mocking, API header format.
- Token format validation (ghu_, ghp_, github_pat_, invalid)
- localStorage save/retrieve/clear
- ghHeaders() construction
- Fallback on 404 from copilot_internal

### `calendar.test.js`
**Tests:** Calendar logic, dayoff counting, weekend handling.
- countWeekendsInMonth() for all month lengths (28вЂ“31 days)
- calToggleDay() toggle mechanics
- countEffectiveDayoffs() (no double-count)
- renderCalGrid() styling (today, weekends, dayoffs)

### `uiHelpers.test.js`
**Tests:** Formatters, sync helpers, HTML escaping.
- fmt1(), fmt2(), fmtInt() with edge cases (0, large numbers, decimals)
- escHtml() with `&<>"` chars
- syncUsage() / syncUsageFromInput() (mock DOM elements, verify sync)

### No E2E Tests (for now)
- Unit tests cover the core logic and critical helpers
- Manual testing in browser during dev
- E2E (Playwright) can be added later if coordination bugs emerge
